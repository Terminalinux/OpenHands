# Автоматическое сохранение изменений в Codespace

## Что настроено

✅ **Автоматический коммит** - каждые 5 минут неиспользуемое время
✅ **Автоматический push** - все коммиты сразу пушатся в репозиторий
✅ **Сохранение при отключении** - pre-push hook гарантирует сохранение перед отключением
✅ **Восстановление при повторном входе** - git автоматически загружает последние изменения

## Как использовать

### Автоматическое сохранение (по умолчанию)

Просто работайте как обычно. Все изменения будут:
1. Автоматически коммититься каждые 5 минут
2. Пушиться в репозиторий
3. Сохраняться при отключении

### Ручное сохранение

Если нужно сохранить срочно, используйте команду:

```bash
save
```

Это создаст коммит с именем "Manual save: [время]" и сразу его запушит.

### Проверка статуса

```bash
git-status-quick
```

Показывает все измененные файлы.

### Полный статус git

```bash
git status
```

## Что сохраняется

Сохраняются все файлы, не указанные в `.gitignore`, включая:
- ✅ Изменения в коде
- ✅ Новые файлы (если не в .gitignore)
- ✅ Конфигурационные файлы
- ❌ `node_modules/` и другие большие папки (в .gitignore)
- ❌ `.env` файлы с секретами (в .gitignore)

## Восстановление при новом сеансе

1. При открытии codespace git автоматически загружает последние изменения
2. Все локальные коммиты восстанавливаются
3. Если были локальные несохраненные изменения - они будут потеряны

## Отключение автосохранения (если нужно)

```bash
export GIT_SKIP_AUTOSAVE=1  # Отключить автосохранение
```

Или удалить git hooks:
```bash
rm -f .git/hooks/pre-push
```

## Логи

Логи автосохранения находятся в:
```
~/.openhands-auto-save.log
```

## Требования

- Необходимо иметь доступ к GitHub (SSH ключ или HTTPS токен)
- `.git/hooks/` должен быть доступен на запись
